Rule Engine Plugin

Description

The Rule Engine Plugin allows to run business rules against each row of data. For the maintenance of the business rules and actions the Business Rules Maintenance Tool - a web application - is used. Once the business logic is complete, the relevant project is exported from the tool in form of a zip file. The zip file is then referenced in the "Rules File Name" field.

Note: make sure that inside the project in the Business Rules Maintenance Tool you define fields that correspond to the names of the fields in the transformation.

The Business Rules Maintenance Tool can be downloaded as a Web Archive (war file) here:: Business Rules Maintenance Tool. The database MySQL/Mariadb database schema required by the tool is available here: Business Rules Maintenance Tool.

Options

The table below describes the options available for the Fixed File Input step:

Option Description
Rules File Name The name of the project zip file. Project zip files are created with the Business Rules Maintenance Tool - a web application to orchestrate rules and actions. You may use variables.
Main Output Step Select the step to which the main output of the rule egine will be forwarded.
Rule Results Step Select the step to which the detailed output of the rule engine will be forwarded.
Rule Results Step Output Type Select which type of results will be output to the steo defined in the Rule Results Step.

Metadata Injection Support

You can use the Metadata Injection supported fields with ETL Metadata Injection step to pass metadata to your transformation at runtime. The following Option and Value fields of the Rule Engine step support metadata injection: Rules File Name and Rule Results Step Output Type
